# 🐛 Bug Fix: 图片尺寸错误 & 游戏突然重启

## 问题描述

### 问题1：图片尺寸不正确 ❌

**症状**：
- 用户上传的图片比例是正确的
- 但游戏中显示的尺寸不对
- 所有物体被强制缩放到相同大小
- 失去了原始图片的比例和设计意图

### 问题2：游戏突然重启 ❌

**症状**：
- 打着打着，快破坏完物品了
- 游戏突然重新开始
- 所有进度丢失
- 非常影响体验

## 根本原因

### 原因1：强制尺寸缩放

**原来的代码**：
```typescript
// ❌ 强制所有物体缩放到屏幕高度的25%
const maxHeight = Math.max(height * 0.25, 120);
if (obstacle.height > maxHeight) {
  obstacle.setScale(maxHeight / obstacle.height);
}
```

**问题**：
- 不管原始图片多大，都缩放到25%屏幕高度
- 破坏了用户上传图片的原始比例
- 小物体被放大，大物体被缩小
- 失去了图片的设计意图

**例子**：
```
用户上传: football.png (100x100像素, 1:1比例)
屏幕高度: 800px
强制缩放: 100 → 200px (800 * 0.25)
结果: 足球被放大2倍 ❌

用户上传: studydesk.png (300x400像素, 3:4比例)
屏幕高度: 800px
强制缩放: 400 → 200px (800 * 0.25)
结果: 书桌被缩小一半 ❌
```

### 原因2：掉出屏幕自动重启

**原来的代码**：
```typescript
// ❌ 玩家掉出屏幕就重启游戏
if (this.player.y > this.scale.height + 100) {
  this.scene.restart();  // 重新开始！
}
```

**为什么会掉出屏幕？**
1. 攻击时的位置锁定不够强
2. 物理引擎的累积误差
3. 碰撞反作用力
4. 屏幕震动影响物理计算
5. 多次攻击后，玩家Y坐标逐渐偏移

**触发条件**：
- 连续攻击5-10次
- 每次攻击玩家Y可能偏移1-2像素
- 累积10-20像素后
- 如果玩家靠近屏幕底部，就会掉出
- 触发 `y > height + 100` 条件
- **游戏重启！**

## 修复方案

### 修复1：保持原始图片尺寸和比例 ✅

```typescript
// ✅ 保持原始图片尺寸（用户已上传正确比例）
// 只在极端情况下缩放（安全检查）
const maxSafeHeight = height * 0.8; // 最大80%屏幕高度
if (obstacle.height > maxSafeHeight) {
  const scale = maxSafeHeight / obstacle.height;
  obstacle.setScale(scale);
  console.log(`⚠️ 图片过大，缩小到 ${(scale * 100).toFixed(0)}%`);
}
// 否则保持原始尺寸！
```

**修改位置**：
1. **玩家尺寸** - `maxPlayerHeight = height * 0.5` (从28%改为50%，只在超大时缩放)
2. **障碍物尺寸** - `maxSafeHeight = height * 0.8` (从25%改为80%，只在超大时缩放)
3. **转换后的物品** - 同样保持原始尺寸

**效果**：
- 保留用户上传图片的原始比例
- 小物体保持小
- 大物体保持大
- 符合设计意图
- 视觉层次更丰富

### 修复2：掉出屏幕时恢复位置而不是重启 ✅

```typescript
// ✅ 掉出屏幕时重置位置，而不是重启游戏
if (this.player.y > this.scale.height + 100) {
  console.warn('⚠️ 玩家掉出屏幕，重置位置');
  this.player.y = this.scale.height - 150; // 安全位置
  this.player.x = Math.max(this.player.x, 100); // 确保不太靠左
  
  if (this.player.body) {
    this.player.body.position.y = this.player.y;
    this.player.body.position.x = this.player.x;
    this.player.setVelocity(0, 0); // 停止下落速度
  }
}
```

**作用**：
- 检测到玩家掉出 → 重置到安全位置
- 不会丢失游戏进度
- 玩家可以继续游戏
- 只是位置调整，没有惩罚

### 修复3：加强攻击时的位置锁定 ✅

```typescript
// 🎯 持续位置锁定（每50ms一次，持续400ms）
const positionLockInterval = this.time.addEvent({
  delay: 50,
  repeat: 7, // 8次 = 400ms
  callback: () => {
    this.player.x = playerOriginalX;
    this.player.y = playerOriginalY;
    if (this.player.body) {
      this.player.body.position.x = playerOriginalX;
      this.player.body.position.y = playerOriginalY;
      this.player.setVelocity(0, 0);
    }
  }
});
```

**作用**：
- 攻击期间每50ms强制恢复位置
- 防止物理引擎推开玩家
- 防止累积误差
- 确保位置绝对稳定

## 尺寸设计哲学

### 修复前的问题
```
所有物体 → 强制25%屏幕高度
  ├─ 足球 (小) → 被放大
  ├─ 箱子 (中) → 被调整
  └─ 书桌 (大) → 被缩小
结果：失去层次感，不符合设计 ❌
```

### 修复后的设计
```
保持原始尺寸 + 比例
  ├─ 足球 (小) → 保持小 → 易打破
  ├─ 箱子 (中) → 保持中 → 中等难度
  └─ 书桌 (大) → 保持大 → 更多血量
结果：视觉层次丰富，符合设计 ✅
```

### 用户体验提升

**视觉层次**：
- 小物体 vs 大物体 → 明显区别
- 视觉更有趣
- 更有挑战性

**设计意图保留**：
- 用户上传什么比例 → 就显示什么比例
- 尊重原始设计
- 不做不必要的调整

**游戏性提升**：
- 大物体 = 更高血量 = 更难打
- 小物体 = 低血量 = 容易打
- 自然的难度分级

## 安全缩放边界

虽然保持原始尺寸，但仍有安全检查：

| 对象 | 安全边界 | 原因 |
|------|----------|------|
| **玩家** | 最大50%屏幕高度 | 太大会遮挡视野 |
| **障碍物** | 最大80%屏幕高度 | 太大无法显示 |
| **转换后物品** | 最大80%屏幕高度 | 保持一致 |

**只有在超过边界时才缩放**：
- 99%的情况下：保持原始尺寸
- 1%的极端情况：缩放到安全大小
- 保护游戏不崩溃

## 技术细节

### 为什么是50ms间隔？

```
游戏帧率: 60 FPS = 16.67ms/帧
物理更新: 约60Hz = 16.67ms/次
位置锁定: 50ms = 3帧/次

理由：
- 不能每帧锁定（太频繁，影响性能）
- 不能太慢（累积误差）
- 50ms = 3帧 = 平衡点
```

### 为什么是400ms总时长？

```
攻击动画时长:
├─ 锤子动画: 200ms (wind-up + swing)
├─ 震动效果: 150ms
└─ 粒子效果: 100ms
────────────
总计: 450ms

位置锁定: 400ms (覆盖大部分)
每次: 50ms
总次数: 8次
```
